# ⏱️ Daily 10min Quiz

1回10分程度で解ける学習クイズアプリです。カテゴリを選ぶと、そのジャンルの問題プールからランダムに10問出題されます。スマホでの利用を想定したモバイルファーストのデザインです。

## 収録カテゴリ

| カテゴリ | 内容 |
| --- | --- |
| 🧱 オブジェクト指向 | カプセル化・継承・ポリモーフィズムなどOOPの基礎(全16問のプールから10問出題) |
| 🧩 デザインパターン | GoFデザインパターンの目的・使いどころ(全16問のプールから10問出題) |
| 🏛️ SOLID原則 | オブジェクト指向設計5原則の理解(全16問のプールから10問出題) |
| 🎯 クリーンアーキテクチャ | レイヤー構成と依存性のルールの理解(全16問のプールから10問出題) |
| 🤖 バイブコーディング | AIにコードを書かせる開発スタイルの考え方と実践(全16問のプールから10問出題) |

## 機能

- カテゴリ選択 → 「クイズに挑戦」or「概要を学ぶ」を選択
- クイズ: 10問出題 → 結果・ふりかえり表示
- 概要ページ: カテゴリ内の全原則/パターンの解説を分類ごとに読める(クイズと解説データを共有)
- 回答するとその場で正誤と解説を表示
- 各問題に「詳しい解説」付き:原則/パターンの定義、適用前後のコード、メリット、言語別実装例(Rust / F# / Kotlin / TypeScript)、経済情報ドメイン(企業・従業員)での実装例
- 出題順・選択肢の並びは毎回シャッフル
- 経過タイマー付き(10分を超えると赤字で表示)
- ダークモード対応(OSの設定に追従)

## 開発

ビルド不要の静的サイトです(HTML / CSS / Vanilla JS)。ローカルで確認するには:

```bash
# リポジトリ直下で任意の静的サーバーを起動
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

### ディレクトリ構成

```
├── index.html          # エントリポイント(全画面を含むSPA)
├── css/style.css       # スタイル(モバイルファースト)
├── js/app.js           # アプリロジック(画面遷移・出題・採点)
├── data/questions.js   # 問題データ(カテゴリ・問題プール)
├── data/concepts.js    # 詳しい解説データ(コンセプト単位)
└── .github/workflows/deploy.yml  # GitHub Pages自動デプロイ
```

## 問題・カテゴリの追加方法

`data/questions.js` を編集するだけで追加できます。

1. カテゴリを増やす場合は `categories` 配列にエントリを追加
2. `questions` オブジェクトにカテゴリIDをキーとして問題配列を追加

```js
{
  id: "xx-01",
  conceptId: "singleton", // data/concepts.js の詳しい解説キー(省略可)
  question: "問題文",
  code: "任意のコードスニペット(省略可)",
  choices: ["選択肢1", "選択肢2", "選択肢3", "選択肢4"],
  answerIndex: 0,        // choicesの正解インデックス(表示時はシャッフルされる)
  explanation: "解説文",
}
```

詳しい解説(定義・適用例・メリット・言語別実装例・ドメイン例)は `data/concepts.js` にコンセプト単位で定義し、問題側の `conceptId` から参照します。同じコンセプトの問題は解説を共有します。

## デプロイ

`main` ブランチに push すると、GitHub Actions(`.github/workflows/deploy.yml`)が GitHub Pages へ自動デプロイします。

### 初回セットアップ(1回だけ必要)

リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。
